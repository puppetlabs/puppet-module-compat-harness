# Known Incompatible Modules

This document lists modules that have been tested against Puppet Core and determined to be incompatible or partially incompatible. Modules marked **Incompatible** are excluded from the active compatibility test suite. Modules marked **Partial** remain in the test suite but carry a documented limitation.

## Incompatibility Summary

| Module | Puppet Core Tested | Status | Reason | Recommended Replacement | Details |
|--------|-------------------|--------|--------|------------------------|---------|
| [garethr-docker](https://github.com/garethr/garethr-docker) | 8.19.0 | Incompatible | Unmaintained module and legacy dependency graph incompatible with current Ruby/Puppet Core test toolchain | N/A | Repository is stale (last commit approximately 9 years old). Under Puppet Core 8.19.0 profile (Ruby 3.2), bootstrap resolves legacy dependencies (for example rake 10.5.0 and older beaker stack) that fail native extension build steps with `NoMethodError` in rake internals while building transitive gems (`unf`, `rainbow`). Result is persistent harness bootstrap failure and no reliable compatibility signal for modern Puppet Core. |
| [puppet-choria](https://github.com/choria-io/puppet-choria.git) | N/A | Incompatible | OpenVox-only module; hard failure for Puppet Core and Puppet Enterprise users | N/A | As of version 0.32.0, puppet-choria explicitly targets OpenVox only. The module will cause a hard failure when used with Perforce Puppet installations, making it incompatible with this test harness. |
| [puppet-r10k](https://github.com/voxpupuli/puppet-r10k) | 8.19.0 | Partial | The `r10k::mcollective` class depends on the choria fixture, which hard-fails on Puppet Core with "Choria only supports OpenVox" | N/A | The core r10k functionality (deployment, hooks, configuration management) is compatible with Perforce Puppet products and passes all relevant unit and acceptance tests. The `r10k::mcollective` integration class — which wires r10k into a Choria/MCollective orchestration layer — cannot be used with Puppet Core because the choria module (as of version 0.32.0) raises a hard error when it detects a non-OpenVox distribution. Puppet Core users should not deploy the `r10k::mcollective` class. The harness treats mcollective-only test failures from this choria check as a compatibility warning rather than a full failure. |
| [puppet-staging](https://github.com/voxpupuli/puppet-staging) | 8.17.0 | Incompatible | Deprecated by Maintainer and uses legacy test framework dependency | [puppet-archive](https://github.com/voxpupuli/puppet-archive#migrating-from-puppet-staging) | Module targets Puppet 5.5.8 - 6.x and uses deprecated `PuppetlabsSpec::PuppetInternals` helper internals in parser function specs (scope_defaults_spec.rb, staging_parse_spec.rb). Modern Puppet Core 8 test toolchain (voxpupuli-test + rspec-puppet) does not provide the PuppetlabsSpec module, resulting in NameError during unit test execution. Module maintainers would need to migrate specs to modern rspec-puppet patterns to support Puppet Core 8. |
